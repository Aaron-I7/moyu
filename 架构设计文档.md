# 摸鱼吧（微放松平台）架构设计文档

## 一、文档概述

### 1.1 文档信息

| 项目名称 | 摸鱼吧（微放松平台） |
|---------|-------------------|
| 文档版本 | v2.0 |
| 编写日期 | 2026-02-26 |
| 文档状态 | 初稿 |

### 1.2 文档目的

本文档旨在定义摸鱼吧平台的整体技术架构，为开发团队提供统一的技术规范和实施指南，确保系统具备高性能、高可维护性和良好的用户体验。

### 1.3 适用范围

本文档适用于摸鱼吧平台的前端开发、测试人员。

### 1.4 项目特性

本项目为**纯前端静态网站**，具备以下特点：
- 无后端服务
- 无用户登录
- 无数据持久化
- 即开即用，用完即走

### 1.5 核心设计原则

- **模块独立**：每个游戏/工具都是独立文件夹，互不依赖
- **配置驱动**：通过配置文件管理模块，方便新增/修改/删除
- **图标可配**：支持图标自定义，方便统一管理
- **完全解耦**：模块之间零耦合，可独立开发、测试、部署

## 二、系统概述

### 2.1 项目定位

摸鱼吧是一个轻量化微放松平台，面向 18-35 岁职场人和学生群体，提供碎片化放松服务（单次使用 5-15 分钟）。

### 2.2 核心目标

- **轻量化**：首屏加载 < 3 秒，资源体积小
- **低门槛**：无需学习成本，即开即用
- **零配置**：无需登录注册，无需保存数据
- **纯静态**：可部署到任意静态服务器或 CDN
- **易扩展**：新增模块只需添加文件夹和配置

### 2.3 功能模块

```
摸鱼吧平台
├── 游戏模块
│   ├── 解压类（消消乐、解压球、捏捏乐、水墨涂鸦）
│   ├── 益智类（拼图、成语接龙、数字填色）
│   └── 养成类（虚拟盆栽、极简养宠）
├── 阅读模块
│   └── 小说阅读器（txt 解析、弹窗隐藏）
└── 工具模块
    ├── 减压工具（白噪音）
    ├── 辅助工具（计时器、壁纸生成）
    └── 工作工具（JSON 格式化、时间戳转换、GUID 生成）
```

## 五、模块注册系统

### 5.1 模块配置规范

每个模块必须有 `config.ts` 文件，定义模块元数据：

```typescript
// src/modules/games/xiaoxiaole/config.ts
import type { ModuleConfig } from '@/core/module/types'
import IconXiaoxiaole from './assets/images/icon.svg'

export const xiaoxiaoleConfig: ModuleConfig = {
  id: 'xiaoxiaole',
  name: '消消乐',
  description: '简单有趣的消除游戏，放松心情',
  category: 'games',
  tags: ['解压', '休闲', '消除'],
  icon: IconXiaoxiaole,
  iconType: 'svg',
  route: '/games/xiaoxiaole',
  component: () => import('./index.vue'),
  meta: {
    title: '消消乐 - 摸鱼吧',
    keywords: ['消消乐', '消除游戏', '休闲游戏']
  },
  enabled: true,
  order: 1
}

export default xiaoxiaoleConfig
```

### 5.2 模块类型定义

```typescript
// src/core/module/types.ts
import type { Component } from 'vue'

export type ModuleCategory = 'games' | 'tools' | 'reading'

export type IconType = 'svg' | 'iconify' | 'image' | 'component'

export interface ModuleMeta {
  title: string
  keywords?: string[]
  description?: string
}

export interface ModuleConfig {
  id: string
  name: string
  description: string
  category: ModuleCategory
  tags?: string[]
  icon: string | Component
  iconType: IconType
  route: string
  component: () => Promise<Component>
  meta?: ModuleMeta
  enabled?: boolean
  order?: number
  [key: string]: unknown
}

export interface ModuleRegistry {
  games: ModuleConfig[]
  tools: ModuleConfig[]
  reading: ModuleConfig[]
  all: ModuleConfig[]
}
```

```typescript
    games: [],

    reading: []
  }

  register(config: ModuleConfig): void {
    if (this.modules.has(config.id)) {
      console.warn(`Module ${config.id} already registered, skipping...`)
      return
    }

    if (config.enabled === false) {
      return
    }

    this.modules.set(config.id, config)
    
    if (config.category && this.categorized[config.category]) {
      this.categorized[config.category].push(config)
      this.categorized[config.category].sort((a, b) => (a.order || 0) - (b.order || 0))
    }
  }

  registerMany(configs: ModuleConfig[]): void {
    configs.forEach(config => this.register(config))
  }

  unregister(id: string): void {
    const config = this.modules.get(id)
    if (config) {
      this.modules.delete(id)
      if (config.category && this.categorized[config.category]) {
        const index = this.categorized[config.category].findIndex(m => m.id === id)
        if (index > -1) {
          this.categorized[config.category].splice(index, 1)
        }
      }
    }
  }

  get(id: string): ModuleConfig | undefined {
    return this.modules.get(id)
  }

  getByCategory(category: ModuleCategory): ModuleConfig[] {
    return this.categorized[category] || []
  }

  getAll(): ModuleConfig[] {
    return Array.from(this.modules.values())
  }

  getRegistry(): ModuleRegistry {
    return {
      games: this.categorized.games,
      tools: this.categorized.tools,
      reading: this.categorized.reading,
      all: this.getAll()
    }
  }
}

export const moduleRegistry = new ModuleRegistryImpl()
export default moduleRegistry
```

### 6.1 动态路由生成

```typescript
// src/core/router/index.ts
import { createRouter, createWebHistory, type RouteRecordRaw } from 'vue-router'
import moduleRegistry from '@/core/module/registry'

function generateModuleRoutes(): RouteRecordRaw[] {
  const routes: RouteRecordRaw[] = []
  const modules = moduleRegistry.getAll()

  const categoryRoutes: Record<string, RouteRecordRaw> = {
    games: {
      path: '/games',
      name: 'Games',
      component: () => import('@/views/GamesView.vue'),
      meta: { title: '游戏' },
      children: []
    },
    tools: {
      path: '/tools',
      name: 'Tools',
      component: () => import('@/views/ToolsView.vue'),
      meta: { title: '工具' },
      children: []
    },
    reading: {
      path: '/reading',
      name: 'Reading',
      component: () => import('@/modules/reading/index.vue'),
      meta: { title: '阅读' }
    }
  }

  modules.forEach(module => {
    const route: RouteRecordRaw = {
      path: module.route.split('/').pop() || module.id,
      name: module.id,
      component: module.component,
      meta: {
        title: module.name,
        ...module.meta
      }
    }

    if (categoryRoutes[module.category] && categoryRoutes[module.category].children) {
      categoryRoutes[module.category].children!.push(route)
    }
  })

  Object.values(categoryRoutes).forEach(route => {
    routes.push(route)
  })

  return routes
}

const baseRoutes: RouteRecordRaw[] = [
  {
    path: '/',
    name: 'Home',
    component: () => import('@/views/HomeView.vue'),
    meta: { title: '首页' }
  },
  {
    path: '/:pathMatch(.*)*',
    name: 'NotFound',
    component: () => import('@/views/NotFoundView.vue'),
    meta: { title: '页面未找到' }
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes: [...generateModuleRoutes(), ...baseRoutes]
})

export default router
```

### 12.1 命名规范
